# モダンToDoアプリ 仕様書

## 1. システム概要
本システムは、タスクおよびカテゴリを管理できるシンプルなWebベースのToDoアプリケーションです。
ユーザーはタスクの追加、完了・未完了の切り替え、削除、期限設定、カテゴリ分類、ドラッグ＆ドロップによる並べ替えを行うことができます。バックエンドは Cloudflare Workers (Hono) を使用し、データは D1 データベース（SQLiteベース）に永続化されます。フロントエンドはシングルページアプリケーション (SPA) として実装されています。

## 2. 起動・アクセス方法
本アプリケーションは、Cloudflare Workers と D1 データベースを使用して動作します。
ローカルで起動・テストする場合は、Wrangler（Cloudflareの開発ツール）を使用します。

1. ターミナル（端末）を開き、プロジェクトのディレクトリに移動します。
2. 以下のコマンドを実行して依存関係をインストールし、ローカルサーバーを起動します。
   ```bash
   ~/.nvm/versions/node/v20.20.1/bin/npm install
   ~/.nvm/versions/node/v20.20.1/bin/npx wrangler dev
   ```
   ※詳細なコマンドパス等は `.clinerules` を参照してください。
3. サーバーが起動したら、ブラウザを開き以下のURLにアクセスしてください。
   - **http://127.0.0.1:8787/** またはターミナルに表示されるURL

## 3. 実行環境・技術スタック
- **バックエンド**: TypeScript, Hono (Cloudflare Workers)
- **フロントエンド**: HTML5, CSS3, JavaScript (Vanilla JS SPA + SortableJS)
- **データストア**: Cloudflare D1 (SQLite)
- **UI/UX**: Google Fonts (Outfit), FontAwesome (Icons)

## 4. データ仕様 (D1 データベース構造)
データは以下のテーブル構造で保存されます (`schema.sql` 参照)。

### 4.1. タスク (`tasks`)
- `id` (INTEGER, PRIMARY KEY, AUTOINCREMENT): タスクの一意なID
- `user_id` (INTEGER, NOT NULL): ユーザー紐付け
- `title` (TEXT, NOT NULL): タスクの名称
- `description` (TEXT): タスクの詳細説明・メモ
- `completed` (BOOLEAN, NOT NULL, DEFAULT 0): 完了状態 (`1`: 完了, `0`: 未完了)
- `deadline` (TEXT): 期限 (フォーマット: `YYYY-MM-DD`, 設定なしの場合はNULL)
- `category_id` (INTEGER): 関連付けられたカテゴリのID (外部キー)
- `display_order` (INTEGER, NOT NULL, DEFAULT 0): 表示順序を管理するための数値

### 4.2. カテゴリ (`categories`)
- `id` (INTEGER, PRIMARY KEY, AUTOINCREMENT): カテゴリの一意なID
- `user_id` (INTEGER, NOT NULL): ユーザー紐付け
- `name` (TEXT, NOT NULL): カテゴリ名
- `display_order` (INTEGER, NOT NULL, DEFAULT 0): 表示順序を管理するための数値

### 4.3. サブタスク (`subtasks`)
- `id` (INTEGER, PRIMARY KEY, AUTOINCREMENT): サブタスクの一意なID
- `task_id` (INTEGER, NOT NULL): 親タスクのID (外部キー)
- `title` (TEXT, NOT NULL): サブタスクの名称
- `completed` (BOOLEAN, NOT NULL, DEFAULT 0): 完了状態
- `display_order` (INTEGER, NOT NULL, DEFAULT 0): 表示順序

## 5. 画面・UI仕様
本アプリケーションは SPA（シングルページアプリケーション）として構築されており、画面遷移を伴わずにビューの表示・非表示を切り替えます。

### 5.1. メインビュー (タスク一覧画面)
- **機能**:
  - カテゴリごとのタスク絞り込み表示 (タブによる切り替え)。
  - タスクの新規追加 (タイトル、カテゴリ、期限の設定)。
  - 各タスクの完了・未完了のトグル切り替え。
  - 各タスクの削除。
  - ドラッグ＆ドロップによるタスクの表示順変更。
  - **タスク詳細展開**: 各タスクをクリックすることで詳細エリアが展開し、メモの編集やサブタスクの管理が可能。
  - **自動保存**: メモやサブタスク名の変更は自動保存。
  - **完了連動**: すべてのサブタスクを完了すると、メインタスクも完了にするか確認ダイアログを表示。

### 5.2. カテゴリ管理ビュー
- **機能**:
  - 登録済みカテゴリの一覧表示。
  - カテゴリの新規追加・削除。
  - **カテゴリ名の直接編集**: カテゴリ名を直接編集でき、自動保存される。
  - ドラッグ＆ドロップによるカテゴリの表示順変更。
